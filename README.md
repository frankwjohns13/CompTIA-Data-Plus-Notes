# CompTIA Data Plus Notes

## Domain 1.0 - Data Concepts and Environments (20%)

**Explain data concepts**
- **Database types:**
  - **Relational(SQL)** - Data stored in tables with rows and columns; uses keys to link tables (e.g., MySQL, PostgreSQL, Microsoft SQL Server). Best for structured data with relationships.
  - **Non-relational(NoSQL)** — Flexible storage for unstructured or semi-structured data. Types include document (MongoDB), key-value, column-family, and graph databases.
  - **Data warehouses** — Centralized repository for historical data from multiple sources, optimized for analysis and reporting.
  - **Data lakes** Large repository that stores raw data in its native format (structured, semi-structured, unstructured).
- **Data structures:**
  - **Structured** - Organized in a fixed format (rows and columns), easy to query (e.g., Excel spreadsheets, relational databases).
  - Semi-**structured** — Has some organization but not rigid (e.g., JSON, XML, CSV files).
  - **Unstructures** — No predefined format (e.g., text documents, images, videos, emails).
- **File extensions:**
  - .cvs — Comma-Separated Values
  - .json — JavaScript Object Notation
  - .xml — Extensible Markup Language
  - .parquet, .avro — Efficient columnar formats for big data
- **Data types:**
  - **Numeric** - Integer, Float/Decimal
  - **Text/String** - Character data
  - **Date/Time** - Temporal data
  - **Boolean** - True/False values
  - **Binary** - Raw data (images, files)

**Identify data sources**
- Databases (SQL, NoSQL)
- APIs and web services
- Website data and logs
- Files and repositories
- Streaming data sources

**Recognize infrastructure concepts**
- Cloud vs On-premise — Scalability, cost, management
- Storage types: Block, File, Object storage
- Containerization: Docker, Kubernetes basics

**Identify data tools**
- Coding environments (Python, R, SQL)
- Business Intelligence (BI) software (Tableau, Power BI)
- Analysis platforms and notebooks (Jupyter)

**Understand AI concepts**
- AI models (supervised, unsupervised, reinforcement learning)
- Natural Language Processing (NLP)
- Robotic Process Automation (RPA)

## Domain 2.0 – Data Acquisition and Preparation (22%)

**Use data acquisition methods**
- **Data Integration** — Combining data from multiple sources
- **Queries** — Using SQL or other languages to extract data
- **ETL/ELT Processes** — Extract, Transform, Load (or Load first then Transform)
- **APIs and Web Scraping** — Pulling data from external services or websites

**Perform data exploration**
- Identify missing values, duplicates, and outliers
- Understand data distribution and patterns
- Use descriptive statistics (mean, median, mode, standard deviation)
- Visualize initial data to spot issues

**Apply data transformation**
- **Cleansing** — Remove or fix bad data
- **Merging** — Combine datasets
- **Parsing** — Break down complex fields
- **Formatting** — Standardize data types and formats
- **Normalization** — Scale data to a common range
- **Aggregation** — Summarize data (sum, average, count)

## Domain 3.0 – Data Analysis (24%)

**Communicate analysis results**
- Choose appropriate methods for different audiences (executives vs technical teams)
- Create clear, actionable insights
- Use storytelling techniques to present findings
- Tailor visualizations and reports to stakeholder needs

**Select statistical methods**
- **Descriptive Statistics** — Mean, median, mode, standard deviation, variance
- **Inferential Statistics** — Hypothesis testing, confidence intervals
- **Correlation and Regression** — Relationships between variables
- **Probability distributions** — Normal, binomial, etc.
- **Sampling techniques** — Random, stratified, cluster sampling

**Troubleshoot analysis issues**
- Identify root causes of inaccurate results
- Validate data sources and assumptions
- Use diagnostic tools and techniques
- Document and resolve analysis problems

## Domain 4.0 – Visualization and Reporting (20%)

**Create effective visuals**
- Choose the right chart type for the data and message
  - **Bar/Column charts** — Comparisons
  - **Line charts** — Trends over time
  - **Pie/Donut charts** — Proportions (use sparingly)
  - **Scatter plots** — Relationships between variables
  - **Heat maps** — Density or magnitude
- Apply design best practices (color, labels, simplicity)
- Avoid misleading visuals (truncated axes, inappropriate scales)

**Deliver reports**
- Static reports (PDF, Excel)
- Interactive dashboards (Tableau, Power BI)
- Self-service reporting
- Scheduled automated reports

**Validate reporting accuracy**
- Cross-check calculations and data sources
- Perform peer reviews
- Test with sample data
- Document assumptions and limitations

## Domain 5.0 – Data Governance (14%)

**Explain data management practices**
- **Data Documentation** — Metadata, data dictionaries, lineage
- **Versioning** — Tracking changes to datasets
- **Data Lineage** — Understanding data flow from source to destination
- **Data Catalogs** — Centralized repositories for discovering data assets

**Summarize compliance requirements**
- Data retention policies
- Audit and logging requirements
- Regulatory compliance (GDPR, CCPA, HIPAA, etc.)
- Industry-specific regulations

**Compare privacy and protection strategies**
- **Access Control** — Role-Based Access Control (RBAC), least privilege
- **Encryption** — At rest and in transit
- **Data Masking / Anonymization** — Protecting sensitive information
- **Data Loss Prevention (DLP)** — Preventing unauthorized data exfiltration

**Implement quality assurance**
- **Data profiling** — Assessing data quality
- Data validation rules
- Monitoring for anomalies and drift
- Data quality dashboards and reporting
